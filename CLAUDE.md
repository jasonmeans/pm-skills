# CLAUDE

Use this repository as a portable skill library.

## Rules

- Prefer `skills/<skill-name>/SKILL.md` over ad-hoc instructions.
- Keep all new skill names in lowercase-kebab-case.
- Keep writing concise, actionable, and reusable.
- Do not add work confidential information, internal links, or company-specific code.

## Skill File Minimum

Each `SKILL.md` must include:
- Purpose
- When to use
- Inputs
- Outputs
- Steps
- Examples
- Non-goals / Guardrails
