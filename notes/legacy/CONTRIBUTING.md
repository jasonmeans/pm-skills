# Contributing

## Branching
- Use short-lived branches: `feat/*`, `chore/*`, `docs/*`
- Keep PRs focused and reviewable

## Commit Quality
- One logical change per commit
- Clear commit messages that explain why

## Documentation Requirement
Any structural or process change must update at least one of:
- `README.md`
- `docs/playbooks/*`
- `docs/adr/*`
